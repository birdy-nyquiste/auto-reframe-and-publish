# 06 — 构建受限且幂等的草稿交付边界

**What to build:** 让已经提交的 canonical 改写产物能够被确定性地转换成只读、可重建的交付请求，通过受限 Blog 适配器上传必要图片并创建发布草稿；同一投稿任务的失败重试不会重复创建草稿，且任何未被外部服务确认的发送都不能被误记为完成。

**Blocked by:** 05 — 安全生成并验证改写产物

**Status:** wontfix

> ADR-0009 已取代本票据。当前即时公开接口没有草稿或服务端幂等键；新的独立发布任务由 Ticket 10 实现，不能通过保留投稿任务第五里程碑来兼容本方案。

- [ ] fake Blog API 场景能够从改写产物生成并验证请求、上传必要图片、映射发布目标并创建发布草稿。
- [ ] 请求携带稳定的投稿任务幂等键；在超时、未知响应或进程中断后的重试不会创建重复草稿。
- [ ] 原始与规范化交付响应被持久保存，只有外部服务确认持久接受且外部标识已保存后，投稿任务才达到完成里程碑。
- [ ] 生成的交付请求是改写产物的只读派生物，可安全重建且不允许 Agent 手工编辑。
- [ ] Blog 客户端能力只包含必要的图片上传和草稿创建，以及真实接口支持时的草稿更新；公开发布、删除、用户管理和部署能力不存在。
- [ ] 未确认请求、拒绝响应、恶意响应内容和幂等重试均有自动化覆盖。
