# 11 — 通过真实端到端验收并达到 `ready`

**What to build:** 让操作人在固定 Windows 主机上使用正式改写规则、受支持的运行 Agent、真实微信输入和真实 Blog API，完成从投稿到安全发布草稿的完整流程，并以审计、恢复和安全证据证明系统已经达到可使用的 `ready`，同时仍把公开发布留在 Blog 管理端人工批准。

**Blocked by:** 08 — 验证 Windows Computer Use 实机链路; 09 — 接入正式改写规范与默认提示词; 10 — 适配真实 Blog API

**Status:** ready-for-agent

- [ ] 在固定 Windows 采集主机上，从真实收件会话中的有效投稿创建真实 Blog 发布草稿，且该草稿等待 Blog 管理员人工批准而不会自动公开。
- [ ] 验收使用明确支持的 Codex 或 Claude Code 运行环境，并验证 canonical Skill 的安装、发现和四个操作入口。
- [ ] 真实链路覆盖默认与自定义改写要求、静态图片、多任务排序、单任务失败隔离、幂等重试和中断恢复。
- [ ] 状态和运行报告能够从任务库重建输入窗口、任务创建、采集证据、改写版本、交付尝试、阻塞原因和外部草稿结果。
- [ ] 安全验收确认不信任来源内容、不使用 OCR 正文、不泄露凭据、不扩大 Blog 能力，并在所有退出路径清空剪贴板。
- [ ] 只有正式改写资源、Windows 实机验收和真实 Blog API 验收全部通过后，readiness 才报告 `ready`。
