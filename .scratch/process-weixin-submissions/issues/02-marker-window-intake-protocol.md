# 02 — 落实微信输入窗口与严格投稿协议

**What to build:** 让操作人用基线和批次标记准确界定每次运行的新消息范围，并把范围内符合严格协议的任务头和相邻公众号文章卡片登记为独立投稿任务；不完整或不支持的输入被明确隔离，系统不猜测归属，也不会把运行期间到达的新消息提前纳入。

**Blocked by:** 01 — 打通一条脚本化投稿纵向链路

**Status:** ready-for-agent

- [ ] initialize 建立基线且不导入更早的微信历史；正常运行只处理相邻批次标记之间的输入窗口。
- [ ] 任务头支持必填发布目标、完全省略或为空的自定义改写要求，以及多行自定义改写要求。
- [ ] 首版只接受任务头后严格相邻的一篇公众号文章卡片，并为未来可选文章数量字段保留兼容解析空间。
- [ ] 未知控制字段、缺少发布目标、缺少相邻来源或不支持的来源类型均产生明确、可审计且不阻塞其他投稿的结果。
- [ ] 内容完全相同的两次投稿仍创建两个不同的投稿任务；当前批次标记之后到达的消息保留到下一次运行。
