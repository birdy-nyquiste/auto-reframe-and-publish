# 04 — 采集完整正文与静态图片证据

**What to build:** 让运行 Agent 能从一篇公众号文章中以复制粘贴取得权威正文文本，以原始顺序保存所有静态来源图片，并留下证明文章已采集完整的不可变采集证据；在微信能力受限时使用受控截图降级，但不依赖 OCR 重建正文，也不臆测未采集的音视频内容。

**Blocked by:** 01 — 打通一条脚本化投稿纵向链路

**Status:** ready-for-agent

**Resolution:** completed

- [x] 正文文本来自复制粘贴，来源 URL 为尽力获取项；缺少 URL 不会在正文、图片和文章结束证据完整时阻塞任务。
- [x] 每个来源图片都保留文章中的出现顺序；相同图片字节可以去重存储，但每次出现仍在清单中独立记录。
- [x] 优先保存原始图片字节，无法取得时可保存从未修改视口截图裁出的静态图片，并记录采集方法与降级信息。
- [x] GIF 只保留静态帧并标记降级；视频和音频不下载、不转录，文字充分的文章带警告继续，媒体为主且文字不足的文章永久失败。
- [x] 原始采集证据不可变，结构化来源能够从原始证据重建并通过完整性检查。
- [x] fixtures 覆盖文章结束识别、重复图片、截图降级、GIF、嵌入式媒体和媒体为主的失败场景。
