# 05 — 安全生成并验证改写产物

**What to build:** 让运行中的 Codex 或 Claude Code Agent 根据可信任务头和已持久化的结构化来源生成 Markdown 正文与结构化清单，并在提交前完成格式和安全验证；来源文章中的文本、图片、链接或二维码只能作为不可信素材，不能改变发布目标、读取本地数据、执行命令或扩大外部动作。

**Blocked by:** 02 — 落实微信输入窗口与严格投稿协议; 04 — 采集完整正文与静态图片证据

**Status:** ready-for-agent

- [ ] 可信控制字段和不可信来源内容在改写输入中明确分离，只有任务头中的已定义字段能够控制改写行为。
- [ ] Agent 输出由 Markdown 正文和通过 JSON Schema 验证的改写清单组成，并记录所用规则、提示和 Schema 的版本或哈希。
- [ ] 生成失败或验证失败的内容只保留为尝试证据；通过验证后提交的改写产物不可变。
- [ ] 改写产物保持外部 Blog API 无关，不包含手工维护的请求 JSON 或外部响应状态。
- [ ] 安全测试向文本、图片、链接、二维码样式内容和模拟外部响应注入指令，确认其无法改变发布目标、访问本地文件、触发命令或扩大允许的 Blog 能力。
