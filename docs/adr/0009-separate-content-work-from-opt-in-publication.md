# 分离内容处理与可选的公开发布

投稿任务只负责从微信输入生成经过校验且不可变的改写产物；公开发布由引用该改写产物的独立发布任务负责。两种任务拥有不同的状态、事件和失败语义，Blog 字段、认证、即时发布响应及重试风险不进入采集、来源重建或改写模块。

操作人启动一次运行时拥有本次发布选择。只有操作人明确选择自动发布，运行才会为本次成功达到改写产物完成的投稿任务创建并执行发布任务；未提及发布和明确选择不发布具有相同语义，均不调用 Blog 写接口。发布选择是运行级可信控制，不是微信任务头字段，来源内容、图片、链接和外部响应都不能改变它。

当前 Blog 写接口会立即公开文章，因此发布任务的成功结果是公开文章而不是发布草稿。适配器允许创建文章和使用只读接口确认结果，不实现删除、用户管理或部署。接口没有服务端幂等时，发布任务必须在发送前持久化固定请求和 slug；无法确认结果时进入未知结果阻塞，不得自动重复发送。

该选择保留单一端到端 Skill，但其编排可以在改写产物完成后停止，也可以在同一次明确授权的运行中继续执行发布任务。分级就绪仍然适用；正式可用要求在安全测试目标上验证公开发布、未知结果处理和必要清理流程，不能再以草稿目标作为验收条件。

这一决定取代将 Blog 视为外部草稿服务的 ADR-0006，并修正其他文档中把外部交付视为投稿任务第五个里程碑或默认草稿交付的表述。
