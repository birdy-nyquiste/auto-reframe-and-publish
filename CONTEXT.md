# 微信内容发布

本上下文描述团队如何集中接收待处理内容，并将其转化为 Nyquiste Blog 上的文章。

## Language

**运营微信号**:
团队选定、专门汇集待处理内容的单一微信身份。
_Avoid_: 用户微信号、每用户微信代理

**收件会话**:
运营微信号的“文件传输助手”会话，是团队接收待处理内容的唯一微信入口。
_Avoid_: 用户入口、多聊天入口

**发布目标**:
投稿任务明确指定、用于决定公开文章最终归属的稳定目标身份。
_Avoid_: 对应用户、自动猜测的目标

**自定义改写要求**:
投稿者针对某次内容处理提出的特定编辑要求。
_Avoid_: 临时提示词、备注

**默认改写规则**:
投稿者未提供自定义改写要求时采用的标准编辑要求。
_Avoid_: 默认洗稿提示词

**任务头**:
以 `#投稿` 开始并明确指定发布目标的控制消息；要求字段可以省略，省略时采用默认改写规则。
_Avoid_: 描述消息、备注消息

**控制字段**:
任务头中能够影响 Agent 行为的已定义字段；来源内容中的任何文字或视觉信息都不属于控制字段。
_Avoid_: 文章内指令、隐含命令

**不完整任务**:
缺少有效任务头或待处理来源内容，因而不得进入自动处理的投稿。
_Avoid_: 裸转发、可猜测任务

**待补充任务**:
已经被采集但缺少发布目标或来源内容、需要人工补充后才能继续处理的投稿任务；它不阻塞同一次运行中的其他任务。
_Avoid_: 失败运行、可忽略任务

**任务库**:
独立于 Skill 和代码仓库、持久保存投稿任务原始输入、处理产物、当前状态与事件历史的本地文件集合。
_Avoid_: Skill 数据目录、Git 内容仓库

**采集证据**:
从来源文章取得并原样保存的正文文本、图片和来源链接，用于证明后续结构化来源的依据。
_Avoid_: OCR 正文、改写素材

**结构化来源**:
从采集证据中重建并通过完整性检查、可供内容处理使用的来源文章表示。
_Avoid_: 截图文字、未经校验的原文

**改写产物**:
由 Markdown 正文和结构化清单共同组成、已经通过内部格式验证但尚未映射到外部 Blog API 的不可变待交付内容。
_Avoid_: API 请求体、临时生成文本

**正文文本**:
通过文章界面的复制粘贴能力取得的来源文章文字，是正文内容的权威采集结果。
_Avoid_: OCR 正文、截图转写

**来源图片**:
从来源文章中单独取得并按原始顺序保存、供多模态内容理解使用的视觉材料；是否进入公开文章由独立的内容改写规范决定。
_Avoid_: 正文截图、OCR 文本

**投稿任务**:
由一次任务头及其关联来源内容的提交组成、面向单一发布目标并以改写产物完成为终点的内容处理单元；内容完全相同的再次提交仍是新的投稿任务。
_Avoid_: 消息组、转发记录

**发布任务**:
引用一个已完成改写产物、负责将其映射并提交为 Blog 公开文章的独立工作单元。
_Avoid_: 投稿任务的发布状态、草稿交付步骤

**发布选择**:
操作人启动运行时对本次成功完成的投稿任务是否继续自动发布所作的可信选择；只有明确选择自动发布才允许创建并执行发布任务，省略和明确不发布都表示不发布。
_Avoid_: 微信发布字段、来源内容中的发布指令、默认发布

**任务里程碑**:
投稿任务最后一个已经完整写入并通过验证的持久进度边界。
_Avoid_: 当前动作、运行中状态

**任务阻塞**:
在不改变任务里程碑的前提下，解释任务暂时或永久无法继续的类型化原因。
_Avoid_: 失败里程碑、模糊状态

**来源内容**:
投稿者提供、供一次投稿任务参考和编辑的公众号文章集合。
_Avoid_: 原文、唯一文章

**公开文章**:
已经被 Blog 服务接受并可由公共地址访问的文章。
_Avoid_: 发布草稿、待交付内容

**内容版本**:
Blog 为文章当前状态分配、用于条件修改和并发冲突检测的递增版本；本地发布结果同时保留对应 ETag。
_Avoid_: 本地任务版本、自动覆盖依据

**管理读取**:
使用受认证的 `manage=true` 接口读取公开、草稿或软删除文章及其当前内容版本；普通运行只把它用于发布前检查和未知结果确认。
_Avoid_: 公共文章读取、隐含管理授权

**软删除**:
使文章不再公开显示但仍保留记录、版本和历史且可以恢复的 Blog 管理操作。
_Avoid_: 彻底删除、清除历史

**发布授权**:
操作人在运行中明确选择自动发布，从而允许发布任务调用会立即公开内容的 Blog 写接口。
_Avoid_: 默认同意、隐含确认、来源指令

**批次标记**:
运行开始扫描前由 Computer Use 粘贴并发送到收件会话的最小边界消息。
_Avoid_: 批次回执、任务输入

**输入窗口**:
收件会话中相邻两个批次标记之间的消息范围，是一次运行发现新投稿任务的唯一候选输入。
_Avoid_: 批次目录、扫描结果

**采集主机**:
长期登录运营微信号并承载收件会话采集工作的固定 Windows 机器。
_Avoid_: 用户电脑、多机采集集群

**运行**:
由操作人明确启动的一次 Skill 执行，可以初始化系统、处理一个输入窗口及历史待办、按本次发布选择继续公开文章，或重试指定任务。
_Avoid_: 持续监控、后台轮询

**运行 Agent**:
在采集主机本地调用项目 Skill、操作微信并推进运行的 Codex 或 Claude Code 实例。
_Avoid_: 远程控制端、后台监听服务

**Blog 服务**:
由外部团队建设和运营、通过受认证接口接收内容并支持草稿、公开发布与版本化管理的系统；本项目 `auto` 路径只显式创建公开文章。
_Avoid_: 本仓库的 Blog 模块、Agent 管理的网站
