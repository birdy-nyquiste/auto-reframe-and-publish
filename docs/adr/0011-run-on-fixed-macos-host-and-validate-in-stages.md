# 在固定 macOS 主机上人工触发运行并分级验收

当前这台 macOS 机器取代原计划中的 Windows 主机，成为第一阶段唯一受支持的采集主机。运营微信号长期登录此机，Codex 在本地由操作人显式触发，通过 Computer Use 操作文件传输助手；项目仍不采用轮询、持续监听、远程桌面或多机并发采集。

Computer Use 负责微信导航、批次标记粘贴发送、正文复制、静态图片采集和剪贴板清理。文章采集保持在微信 App 内：正文通过聚焦后全选复制取得，静态图片按出现顺序右键并优先选择 `保存图片`，不通过浏览器抓取替代。确定性脚本只接受经过严格验证的 `macos_computer_use_v1` 输入窗口，负责协议解析、证据落盘、任务推进、Agent 改写结果校验和可选发布。两者之间使用任务库外或 `tmp/` 下的临时 captured-window JSON；微信保存的图片只允许从当前任务库 `tmp/` 暂存区导入，不允许 Agent 手改正式任务、改写、发布或报告记录。

项目 readiness 调整为 `core_validated`、`macos_validated` 和 `ready`：

- `core_validated`：公共 CLI、Schema、文件任务库、恢复/重试、受限 Agent 生成器和 Blog 适配器通过自动化测试；
- `macos_validated`：当前 Mac 上的真实微信、Computer Use、复制正文、静态图采集、批次窗口和退出清理通过受监督验收；
- `ready`：在 `macos_validated` 基础上，默认改写资源、受控真实 Blog 发布和完整端到端场景全部验收通过。

默认内容改写提示词可以先采用明确版本的项目默认值并在后续迭代中升级；每个历史产物继续记录当时使用的文件路径与哈希。正式内容改写规范仍是独立资源，不因临时提示词可用而自动视为已经批准。
