# 03 — 让运行可恢复、可审计、可重试

**What to build:** 让每个运行和投稿任务以持久、类型化、可追溯的方式记录进度与阻塞原因，使中断后的再次运行能够从最后一个完整里程碑继续，暂时性故障受到重试预算约束，耗尽后的任务只有通过显式 retry 才重新进入队列，而 status 始终可以安全只读查看。

**Blocked by:** 01 — 打通一条脚本化投稿纵向链路

**Status:** ready-for-agent

**Resolution:** completed

- [x] 四个投稿任务语义里程碑以原子提交的方式表示已经可靠完成的最后进度；发布使用独立聚合与状态机，活动动作只记录为带运行关联的尝试事件。
- [x] 待补充、等待重试、重试耗尽和永久失败均使用相互区分的类型化任务阻塞，并通过 Schema 拒绝非法组合或状态跃迁。
- [x] 从每个投稿里程碑模拟中断后，下一次允许的操作都能恢复且不会重复提交已确认的副作用。
- [x] 重试策略按操作和错误类别集中配置；预算耗尽后普通 run 不再执行该任务，显式 retry 可以重新启用它。
- [x] 所有可变操作受单写者锁保护，疑似陈旧的锁只报告不自动夺取；status 在存在写运行时仍保持只读。
- [x] 运行、任务、事件、尝试和报告之间保留足够关联，可重建任务创建、失败、重试和完成历史。
